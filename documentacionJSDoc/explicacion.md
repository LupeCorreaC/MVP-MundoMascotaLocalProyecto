Documentación del proyecto con JSDoc
 ¿Qué se hizo?

En este proyecto se implementó documentación del código utilizando **JSDoc**, con el objetivo de mejorar la comprensión, mantenimiento y claridad del sistema "Mundo Mascota Local".

---

¿Dónde se aplicó?

La documentación se agregó directamente en los archivos JavaScript principales del proyecto, específicamente en la rama `registro`, sin modificar la lógica del sistema.

Archivos documentados:

- login.js
- registro.js
- navbar.js
- otros módulos principales del sistema

---

## ¿Cómo se hizo?

Se utilizaron comentarios especiales de JSDoc en formato:

/**
 * Descripción de la función o bloque de código
 * @param {tipo} nombre - descripción
 * @returns {tipo} descripción
 */

Estos comentarios se colocaron encima de funciones o bloques de código importantes para explicar su funcionamiento.

---

## ¿Qué se logró?

- Mejor comprensión del código
- Documentación clara de funciones clave
- Separación lógica sin alterar funcionamiento
- Mejora en la mantenibilidad del sistema

---

## ¿Cómo se puede ver?

La documentación puede visualizarse directamente en los archivos `.js` del proyecto dentro de la rama `registro`.

También se puede verificar en GitHub en los archivos modificados, donde aparecen los comentarios JSDoc agregados.

---

## Conclusión

El uso de JSDoc permitió mejorar la calidad del código sin afectar su funcionamiento, cumpliendo con buenas prácticas de desarrollo y documentación técnica.
